Ce sunt structurile de date

Structurile de date sunt modalitati de organizare si stocare a datelor in memoria calculatorului.

De ce sunt importante?

- Permit stocarea mai multor valori sub un singur nume

- Faciliteaza operatiile asupra colectiilor de date

- Fac programele mai eficiente si mai usor de inteles

Tabloul (vectorul) este cea mai simpla structura de date:

- Stocheaza o colectie de elemente de acelasi tip

- Elementele sunt accesate prin index (pozitie)

- Indexarea incepe de la 0 (primul element are indexul 0)

Exemplu: note = [8, 9, 7, 10] unde note[0] = 8, note[1] = 9, etc.

Care este indexul primului element dintr-un tablou (vector)?
A 0
B 1
C -1
üí° In majoritatea limbajelor de programare, numerotarea pozitiilor incepe de la 0.
2

Operatii cu tablouri

Operatii de baza cu tablouri:

1. Accesare element: tablou[index] - citeste valoarea de la pozitia data

  Exemplu: v = [5, 10, 15]; x = v[0] // x devine 5

2. Modificare element: tablou[index] = valoare_noua

  Exemplu: v[1] = 20 // tabloul devine [5, 20, 15]

3. Adaugare element: se adauga la sfarsitul tabloului

4. Stergere element: se elimina si se rearanjeaza elementele

5. Lungimea tabloului: lungime(tablou) sau len(tablou)

Atentie: Accesarea unui index care nu exista cauzeaza eroare! Daca tabloul are 3 elemente, indexurile valide sunt 0, 1, 2.

Avem tabloul v = [5, 10, 15]. Dupa v[1] = 20, ce contine tabloul?
A [20, 10, 15]
B [5, 20, 15]
C [5, 10, 20]
üí° Elementul v[1] este al doilea element din tablou (indexarea incepe de la 0).
3

Cautarea in tablouri

Cautarea este procesul de gasire a unui element in tablou.

1. Cautarea secventiala (liniara):

- Parcurgem elementele unul cate unul

- Ne oprim cand gasim elementul cautat

- Functioneaza pe orice tablou (sortat sau nu)

- Complexitate: O(n) - in cel mai rau caz parcurgem tot tabloul

Algoritm:

PENTRU i DE LA 0 LA lungime(tablou)-1 EXECUTA

  DACA tablou[i] = valoare_cautata ATUNCI

    returneaza i // am gasit la pozitia i

returneaza -1 // nu am gasit

2. Cautarea binara - mai rapida, dar necesita tablou sortat (vom vedea in atomul urmator).

Cautarea secventiala parcurge elementele:
A De la sfarsit spre inceput
B Unul cate unul, de la inceput
C Aleatoriu
üí° Secvential inseamna in ordine, pas cu pas, de la primul element.
4

Sortarea tablourilor

Sortarea ordoneaza elementele crescator sau descrescator.

Bubble Sort (sortarea prin interschimbare):

- Compara elemente vecine si le interschimba daca sunt in ordine gresita

- Repeta pana cand nu mai sunt interschimbari necesare

Exemplu: [5, 2, 8, 1] -> sortare crescatoare

- Pas 1: [2, 5, 1, 8] (comparam 5-2, 5-8, 8-1)

- Pas 2: [2, 1, 5, 8]

- Pas 3: [1, 2, 5, 8] - sortat!

Algoritm:

REPETA

  schimbat = false

  PENTRU i DE LA 0 LA n-2 EXECUTA

    DACA tablou[i] > tablou[i+1] ATUNCI interschimba; schimbat = true

PANA CAND schimbat = false

Sortarea Bubble Sort compara:
A Primul element cu ultimul
B Elemente vecine (consecutive)
C Toate elementele simultan
üí° Bubble = bula. Elementele mai mari "plutesc" spre sfarsit prin comparari succesive cu vecinii.
5

Algoritmi pe tablouri

Algoritmi frecvent utilizati pe tablouri:

1. Gasirea maximului:

  max = tablou[0]

  PENTRU i DE LA 1 LA n-1: DACA tablou[i] > max ATUNCI max = tablou[i]

2. Calcularea sumei:

  suma = 0

  PENTRU fiecare element: suma = suma + element

3. Calcularea mediei:

  media = suma / numar_elemente

4. Numararea elementelor pare:

  contor = 0

  PENTRU fiecare element: DACA element % 2 = 0 ATUNCI contor = contor + 1

5. Inversarea tabloului:

  Interschimbam primul cu ultimul, al doilea cu penultimul, etc.

Sfat: Majoritatea algoritmilor pe tablouri necesita parcurgerea elementelor cu o bucla!

Pentru a gasi cel mai mare element dintr-un tablou, trebuie sa:
A Sortam tabloul si luam ultimul element
B Parcurgem tabloul si comparam fiecare element cu maximul curent
C Ambele variante sunt corecte
üí° Ambele metode functioneaza, dar parcurgerea directa este mai eficienta.
