Ce este o baza de date

Baza de date este o colectie organizata de date stocate electronic.

Componentele principale:

- Tabel: structura care stocheaza datele (similar unui tabel Excel)

- Camp (coloana): un atribut al datelor (ex: Nume, Varsta, Email)

- Inregistrare (rand): un set complet de date pentru o entitate

Cheia primara (Primary Key):

- Este un camp (sau grup de campuri) care identifica unic fiecare inregistrare

- Nu poate avea valori duplicate sau nule

- Exemple: ID_Elev, CNP, Numar_Matricol

Sisteme de gestiune: Microsoft Access, MySQL, SQLite, PostgreSQL.

Ce este cheia primara intr-un tabel?
A O parola pentru baza de date
B Un camp care identifica unic fiecare inregistrare
C Primul camp din tabel
üí° Cheia primara asigura ca fiecare rand din tabel poate fi identificat in mod unic.
2

Tipuri de date in tabele

Fiecare camp dintr-un tabel are un tip de date specific.

Tipuri de date comune:

1. Text (VARCHAR, CHAR)

  - Pentru siruri de caractere: nume, adrese, descrieri

  - Exemplu: "Ion Popescu", "Strada Victoriei 10"

2. Numar intreg (INTEGER, INT)

  - Pentru numere fara zecimale: varsta, cantitate, ID

  - Exemplu: 14, 100, -5

3. Numar real (FLOAT, DECIMAL)

  - Pentru numere cu zecimale: pret, medie

  - Exemplu: 9.75, 123.50

4. Data/Ora (DATE, DATETIME)

  - Pentru date calendaristice si timp

  - Exemplu: 2026-01-20, 14:30:00

5. Boolean (BOOL) - pentru valori Da/Nu (True/False)

Ce tip de date ar trebui folosit pentru a stoca varsta unei persoane?
A Text
B Numar intreg (Integer)
C Data/Ora
üí° Varsta este un numar intreg (ex: 14, 25, 40).
3

Interogari SQL de baza

SQL (Structured Query Language) este limbajul pentru lucrul cu baze de date.

Comanda SELECT - citirea datelor:

Sintaxa: SELECT coloane FROM tabel WHERE conditie;

Exemple:

- SELECT * FROM Elevi; - toate coloanele, toti elevii

- SELECT Nume, Varsta FROM Elevi; - doar coloanele Nume si Varsta

- SELECT * FROM Elevi WHERE Varsta > 14; - elevii cu varsta peste 14

- SELECT * FROM Elevi WHERE Clasa = '8A'; - elevii din clasa 8A

Clauze utile:

- ORDER BY coloana - sorteaza rezultatele

- ORDER BY coloana DESC - sortare descrescatoare

- LIMIT n - returneaza doar primele n rezultate

Ce comanda SQL selecteaza toate coloanele din tabelul Elevi?
A SELECT * FROM Elevi
B GET ALL FROM Elevi
C SHOW Elevi
üí° SELECT este comanda de selectare, * inseamna toate coloanele, FROM specifica tabelul.
4

Relatii intre tabele

Relatiile conecteaza datele din tabele diferite.

Cheia straina (Foreign Key):

- Este un camp care face referinta la cheia primara din alt tabel

- Creeaza o legatura intre tabele

Exemplu: Tabel Elevi si Tabel Clase

Elevi: ID_Elev (PK), Nume, ID_Clasa (FK)

Clase: ID_Clasa (PK), Nume_Clasa, Diriginte

Tipuri de relatii:

- 1:1 (unu la unu) - un elev are un singur carnet

- 1:N (unu la multi) - o clasa are mai multi elevi

- N:M (multi la multi) - un elev are mai multe materii, o materie are mai multi elevi

Beneficii: Evitam duplicarea datelor si pastream consistenta.

Ce este o cheie straina (Foreign Key)?
A O cheie care deschide alt tabel
B Un camp care face referinta la cheia primara din alt tabel
C O cheie primara duplicata
üí° Cheia straina leaga doua tabele, referind cheia primara din tabelul parinte.
5

Operatii INSERT, UPDATE, DELETE

Comenzi SQL pentru modificarea datelor:

1. INSERT - adaugare inregistrare noua:

INSERT INTO Elevi (Nume, Varsta, Clasa) VALUES ('Ana Pop', 14, '8A');

2. UPDATE - modificare date existente:

UPDATE Elevi SET Varsta = 15 WHERE Nume = 'Ana Pop';

Atentie: Fara WHERE, se modifica TOATE randurile!

3. DELETE - stergere inregistrari:

DELETE FROM Elevi WHERE Varsta < 10;

Atentie: Fara WHERE, se sterg TOATE randurile!

Sfaturi importante:

- Foloseste intotdeauna WHERE pentru UPDATE si DELETE

- Testeaza cu SELECT inainte de a modifica/sterge

- Fa backup la date inainte de operatii majore

Ce comanda SQL sterge elevii cu varsta sub 10 din tabelul Elevi?
A REMOVE FROM Elevi WHERE Varsta < 10
B DELETE FROM Elevi WHERE Varsta < 10
C DROP Elevi WHERE Varsta < 10
üí° DELETE este comanda SQL pentru stergerea inregistrarilor dintr-un tabel.
