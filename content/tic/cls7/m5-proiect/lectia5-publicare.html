📚 INVATA

Optiuni de publicare (hosting)

Ce inseamna "hosting"?
Hosting (gazduire) inseamna sa pui fisierele site-ului pe un server conectat la internet, 24/7, astfel incat oricine sa poata accesa site-ul tau.
Analogie: Hosting = un apartament pentru site-ul tau pe internet

🏠 Optiuni gratuite de hosting

GitHub Pages
https://username.github.io
  • 100% gratuit
  • Usor de folosit
  • URL profesional
  • Perfect pentru portofolii
Netlify
https://site-name.netlify.app
  • Gratuit pentru proiecte mici
  • Drag & drop upload
  • HTTPS automat
Vercel
https://site-name.vercel.app
  • Gratuit pentru uz personal
  • Foarte rapid
  • Integrare GitHub

Procesul de publicare:

Site LIVE!
  • Logheaza-te pe GitHub
    Du-te la github.com si intra in cont
  • Creeaza un repository nou
    Click pe "+" (colt dreapta sus) → "New repository"
    IMPORTANT: Numeste-l exact: username.github.io
    (inlocuieste "username" cu username-ul tau)
  • Upload fisierele
    Click "uploading an existing file" → trage toate fisierele HTML si CSS
  • Commit changes
    Scrie un mesaj (ex: "First version") → Click "Commit changes"
  • Activeaza GitHub Pages
    Settings → Pages → Source: "Deploy from a branch" → Branch: main → Save
  • Asteapta 1-2 minute
    Site-ul tau va fi disponibil la: https://username.github.io

URL-ul tau va arata asa:

🔒 https:// username .github.io
💡 Sfaturi importante
  • index.html OBLIGATORIU - Pagina principala trebuie sa se numeasca exact "index.html"
  • Litere mici - Foloseste doar litere mici in numele fisierelor
  • Fara spatii - Inlocuieste spatiile cu "-" (despre-mine.html)
  • Testeaza local inainte - Verifica ca totul functioneaza pe calculator
✅ Checklist inainte de publicare
  • ☐ Toate link-urile functioneaza
  • ☐ Imaginile se incarca corect
  • ☐ Textul e verificat (fara greseli)
  • ☐ Site-ul arata bine pe mobil
  • ☐ index.html exista in folder
